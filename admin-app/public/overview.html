Technical Overview

Aura — Dating Platform

"Find your wavelength."
A depth-first matchmaking platform. Two dedicated React applications sharing one Supabase backend. Members build rich profiles, browse deliberately, and connect through a structured flirt-proposal system — no swiping, no algorithms.
Alpha React + TypeScript Tailwind CSS Supabase Netlify Vite
Applications
Two dedicated interfaces — one shared backend

Aura — User App

aura-dating-platform.netlify.app

The member-facing application. Users log in with a username and password, build a detailed profile with up to 5 photos and a multi-chapter questionnaire, then browse and connect with other members at their own pace.

  • Username + password login (no email required)
  • Multi-step questionnaire with 7 chapters + per-chapter visibility control (public / flirts only)
  • Discover page with filter bar (gender, age group, region, kids, desired relationship)
  • Private Observation List — add profiles without notifying them
  • Flirt proposals with a personal message (max 2 attempts per person)
  • Accept / decline incoming proposals — chat unlocks on mutual acceptance
  • Profile modal shows public questionnaire chapters of other members
  • Up to 5 photos per profile, Supabase Storage, set primary photo
  • How-to guide page explaining the platform mechanics
  • Alpha tester feedback form (category, rating, freeform text)

Aura — Admin Panel

aura-dating-platform-admin.netlify.app  🔒

Private management interface for administrators. Password-protected. Full oversight of members, flirts, messages, and incoming alpha tester feedback.

  • Password-protected entry gate
  • Dashboard with KPI cards and signup trend chart
  • Member management — view, edit, suspend or delete profiles
  • Flirts overview — all active chat connections with dates
  • Messages moderation — read and delete any conversation
  • Feedback inbox — all alpha tester submissions with category filter and rating stats
Authentication
Simple login for multi-user alpha simulation

How authentication works

No real authentication system is used. Each profile has a username and a short plaintext password stored in the database. On login, the app looks up the matching profile and saves the profile ID to the browser's localStorage. Logging out clears the session. This allows the demonstrator to switch between any of the 5 pre-built personas and experience the platform from multiple perspectives.

Scope
What is — and is not — included in this alpha

In scope

✓Username / password login, logout, session via localStorage
✓Rich questionnaire (7 chapters, 30+ fields) with per-chapter visibility
✓Discover with filter bar (gender, age, region, kids, relationship type)
✓Private observation list — silent watchlist before proposing
✓Two-step flirt proposal with personal message (max 2 attempts)
✓Accept / decline proposals — chat unlocks on mutual acceptance
✓Public questionnaire view when browsing other profiles
✓Up to 5 profile photos via Supabase Storage
✓Alpha tester feedback system (user submission + admin inbox)
✓Admin: members, flirts, messages, feedback management
✓Premium dark-mode UI, mobile-responsive, deployed on Netlify

Out of scope (alpha)

✗Real email / OAuth authentication
✗Push or email notifications
✗Real-time chat (WebSocket / Supabase Realtime)
✗AI-powered compatibility matching
✗Video or voice calling
✗Premium subscription or payment
✗Profile identity verification
✗Native mobile application
✗GDPR compliance tooling
✗Matching algorithm (currently: random order)
Deployment & Infrastructure
One repository, two Netlify sites, one Supabase project
Source Control
GitHub
One repository — Manitas-ai/Aura-Dating-Platform. Two sub-folders: /user-app and /admin-app. Auto-deploy on every push to main.
Hosting
Netlify — Free Tier
Two Netlify sites pointing to the same repo with different base directories. SPA routing via netlify.toml redirects.
Database, API & Storage
Supabase
Hosted Postgres with anon REST API (RLS enabled). Storage bucket profile-photos for member images. No custom backend server.
